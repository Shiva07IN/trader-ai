## Step 1 — Set Up Supabase (PostgreSQL)

1. Go to [supabase.com](https://supabase.com) → **New project**
2. Choose region **Southeast Asia (Singapore)** — lowest latency from India
3. Set a strong database password (save it!)
4. Wait ~2 minutes for provisioning

**Get your connection strings:**
- Dashboard → Settings → Database → **Connection String**
- Copy the **URI** (Session mode, port 5432) → `DATABASE_URL_SYNC`
- Change port to `6543` (Transaction mode) → `DATABASE_URL` (prefix with `postgresql+asyncpg://`)

```
# Session mode — for Alembic migrations
DATABASE_URL_SYNC=postgresql://postgres.[ref]:[password]@aws-0-ap-southeast-1.pooler.supabase.com:5432/postgres

# Transaction mode — for FastAPI async (asyncpg)
DATABASE_URL=postgresql+asyncpg://postgres.[ref]:[password]@aws-0-ap-southeast-1.pooler.supabase.com:6543/postgres
```

**Run initial migrations** (from local machine):
```bash
cd apps/api
DATABASE_URL_SYNC=<your-supabase-sync-url> alembic upgrade head
```

## Step 2 — Set Up Upstash Redis

1. Go to [upstash.com](https://upstash.com) → **Create Database**
2. Name: `traderai-redis` | Region: **AP-Southeast-1 (Singapore)**
3. Enable **TLS** (required for `rediss://` connections)
4. Free plan: 10,000 requests/day (sufficient for starter)

**Get connection URLs:**
- Dashboard → your database → **Details** → copy the `Redis URL`

```
REDIS_URL=rediss://:password@endpoint.upstash.io:6379/0
CELERY_BROKER_URL=rediss://:password@endpoint.upstash.io:6379/1
CELERY_RESULT_BACKEND=rediss://:password@endpoint.upstash.io:6379/2
```

## Step 3 — Deploy Backend on Render

### 3a. Connect Repository

1. Go to [dashboard.render.com](https://dashboard.render.com)
2. Click **New** → **Blueprint**
3. Connect your GitHub account
4. Select repo: `Shiva07IN/trader-ai`
5. Render will auto-detect `render.yaml` in root

### 3b. Review Blueprint Services

Render will show 4 services to create:
- ✅ `traderai-api` — Web Service (FastAPI)
- ✅ `traderai-worker` — Background Worker (Celery)
- ✅ `traderai-market-sync` — Cron Job (every 5 min, market hours)
- ✅ `traderai-news-sync` — Cron Job (every 30 min)

Click **Apply** to create all.

### 3c. Set Environment Variables

For `traderai-api` service → Environment tab → add:

| Variable | Value |
|---|---|
| `DATABASE_URL` | `postgresql+asyncpg://...supabase.com:6543/postgres` |
| `DATABASE_URL_SYNC` | `postgresql://...supabase.com:5432/postgres` |
| `REDIS_URL` | `rediss://:password@...upstash.io:6379/0` |
| `CELERY_BROKER_URL` | `rediss://:password@...upstash.io:6379/1` |
| `CELERY_RESULT_BACKEND` | `rediss://:password@...upstash.io:6379/2` |
| `OPENROUTER_API_KEY` | `sk-or-v1-...` |
| `GOOGLE_CLIENT_ID` | From Google Cloud Console |
| `GOOGLE_CLIENT_SECRET` | From Google Cloud Console |
| `NEWS_API_KEY` | From newsapi.org (optional) |

> `SECRET_KEY` and `JWT_SECRET_KEY` are auto-generated by Render (defined as `generateValue: true` in render.yaml)

> Worker/Cron services inherit env vars automatically via `fromService` in render.yaml

### 3d. Verify Deployment

```bash
# Check health endpoint
curl https://traderai-api.onrender.com/health

# Expected response:
{
  "status": "healthy",
  "service": "TraderAI API",
  "version": "1.0.0",
  "environment": "production"
}
```

## Step 4 — Deploy Frontend on Vercel

1. Go to [vercel.com](https://vercel.com) → **New Project**
2. Import repo: `Shiva07IN/trader-ai`
3. Set **Root Directory**: `apps/web`
4. Framework: **Next.js** (auto-detected)

**Environment Variables** (Vercel dashboard → Settings → Environment Variables):

| Variable | Value |
|---|---|
| `NEXT_PUBLIC_API_URL` | `https://traderai-api.onrender.com/api/v1` |
| `NEXTAUTH_URL` | `https://traderai.vercel.app` |
| `NEXTAUTH_SECRET` | Generate: `openssl rand -base64 32` |
| `GOOGLE_CLIENT_ID` | From Google Cloud Console |
| `GOOGLE_CLIENT_SECRET` | From Google Cloud Console |

## Step 5 — Configure Google OAuth

1. Go to [console.cloud.google.com](https://console.cloud.google.com)
2. Create project or select existing
3. APIs & Services → Credentials → **Create Credentials → OAuth 2.0 Client ID**
4. Application type: **Web application**
5. Authorised redirect URIs:
   ```
   https://traderai.vercel.app/api/auth/callback/google
   http://localhost:3000/api/auth/callback/google
   ```
6. Copy Client ID and Secret to both Render and Vercel env vars

## Step 6 — Run Database Migrations

After Render deploys, run migrations via Render Shell or locally:

**Option A — Render Shell** (recommended):
1. Render Dashboard → `traderai-api` → Shell
2. Run: `alembic upgrade head`

**Option B — Local with Supabase URL**:
```bash
cd apps/api
export DATABASE_URL_SYNC="postgresql://postgres.[ref]:[pass]@aws-0-ap-southeast-1.pooler.supabase.com:5432/postgres"
alembic upgrade head
```

## Troubleshooting

### Cold Start (slow first request)
- Upgrade API to **Starter** plan ($7/mo) — stays always-on
- Or add a keep-alive ping using [Better Uptime](https://betteruptime.com) (free)

### Supabase Connection Refused
- Use **Transaction mode (port 6543)** for FastAPI, not direct connection
- Enable **IPv4** in Supabase: Settings → Network → Add `0.0.0.0/0`

### Redis SSL Errors
- Upstash requires TLS — use `rediss://` (double-s), not `redis://`
- Celery: add `broker_use_ssl={"ssl_cert_reqs": None}` if needed

### Celery Worker Not Processing
- Check worker logs: Render Dashboard → `traderai-worker` → Logs
- Ensure `CELERY_BROKER_URL` is set on the worker service
- Worker inherits from API service via `fromService` in render.yaml

### Environment Variables Not Applied
- After adding env vars in Render, click **Manual Deploy** to apply
- Check: Dashboard → Service → Environment tab
